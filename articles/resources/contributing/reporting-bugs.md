# <a name="reporting-bugs"></a>Bogues de création de rapports #

Si vous pensez que vous avez trouvé un bogue dans un composant du kit de développement Quantum, nous aimerions beaucoup apprécier un rapport.
Après tout, quelqu’un d’autre peut éprouver des difficultés avec le même problème ; Nous pouvons nous aider à le résoudre pour tout le monde.
La première étape pour signaler un bogue consiste à examiner les problèmes existants qui se trouvent sur chaque référentiel, car il est tout à fait possible qu’une autre personne ait signalé le même problème.

Si le problème a déjà été trouvé avant, il peut être utile de fournir des informations ou du contexte supplémentaires qui peuvent nous aider à diagnostiquer et à résoudre le problème.
Dans ce cas, n’hésitez pas à participer aux discussions sur le problème.

Si votre bogue n’a pas déjà été signalé, nous vous remercions si vous ouvrez un nouveau problème dans le référentiel pour le composant en question.
Lorsque vous créez un nouveau problème, vous êtes invité à fournir des informations qui sont souvent utiles pour créer des rapports de bogues informatifs :

- Les étapes pour reproduire le problème.
- Ce que vous attendiez
- Que s’est-il passé ?
- Informations sur votre environnement logiciel (p. ex. : système d’exploitation, .NET Core et les versions du kit de développement quantique)

Vous pouvez également [créer une requête de tirage](https://help.github.com/articles/about-pull-requests/) pour corriger le bogue directement, s’il est très simple et ne justifie pas la discussion (par exemple, une faute de frappe).
